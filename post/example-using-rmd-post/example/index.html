In this section of the post we will be looking at the data comparisons within different trees in Cambridge and Boston. The objective we tried to do using this dataset was to figure out how to use R to showcase the different ownership of trees and the

So let’s get started. The first thing we needed to do was to download and install tidyverse. This is to help with the creation of our graph. Once tidyverse was installed we ran this line of code below:

library(tidyverse)

The reason we ran library(tidyverse) instead of just using tidyverse is because in order for R to fully understand that we specifically want to use the functions within tidyverse, we have to call it. Henceforth why we use the library() which is designed to tell R studio what we want to call and use functions of.

After that was finished, we then had to get the dataset which contained the tree data. So we ran these two codes below:

# this chunk is to install the package
install.packages("devtools")
devtools::install_github("harvard-ufds/bosTrees")

library(bosTrees)

# This is to load the data
data(bosTrees)
data(camTrees)

Now here is the fun part. So as stated above, our first objective is to showcase the different ownership within the streets of Cambridge. So now to do this we wrote these lines of code which I will explain just after this:

ggplot(data = camTrees, mapping = aes(x = Longitude,
                                      y = Latitude,
                                      color = Ownership)) +
  geom_point(alpha = 0.05) +
  guides(colour = guide_legend(override.aes = list(alpha = 1)))

Let’s dissect this code line by line. So the reason why we use ggplot() is because it’s the function to help us create the graph we want to portray. But if we just said ggplot(), it would give us a blank graph with nothing on it. Henceforth why we give it parameters.

The first of these parameters being which dataset we want to use within our saved Data. To do this we said data = camTrees, to let R studio know we want that specific dataset.

After that, we use the mapping function so as to let R Studio know just how to plot these different points on the graph (Which is why when plotted they make the various streets of cambridge). In which we set in the aesthetic: the X to be longitude, and the Y to be Latitude. Once all of that is done, we set the colors to be equal to ownership. This way we can tell who owns which trees. But we are not done yet, since we have two more lines to talk about.

Let’s start with the guides() line first. Why this line is extremely important is because we need a type of legend to indicate to us just what these colors mean. Cause let’s say we never created that line. Well you will indeed get a graph that shows different colors, but we still wouldn’t know who owns what because there would be no legend to tell us just what the colors stand for.

Now somebody may ask the question “Well that makes sense. But what does the override.aes=list(Alpha = 1) mean?” Well I will explain both that and what the line above stands for. Now the reason why we said the override function, is because the line above the guide function is a transparency function. Which counts for anything relating to the different ownership colors. So if we were to just run the entire code without having the override in there, we would indeed have a legend, but the colors would be so hard to see. So that’s why we added the override function to keep the color non transparent and easier to see.

With all of that being said, congratulations! You just created your first Graph and learned a bit more about R Studio!. So, we’ve come to this point where we’ve shown plotted data sets using R code and as seen in the post, it’s working as it should. Plots in R code however are only half of the story, now it’s time to interpret this whole thing.

Let’s start off with the 2010 NYC Reagents data. Looking back at the original graphs, you may ask “Why do your graphs look very weird and skewed?”. The answer to that is worth examining the data set itself.

(Show image of improper data set)

What you’re looking at here is a data row where in this case a student got a perfect score of 100. However, looking past that category, all the other individual subject scores appear as “NA” data, and this is just one out of many scenarios that appear like this. Situations like this can easily cause misinterpretation of the data because although the student got a perfect score on the exam overall, none of their subject data recorded, which may contribute to “Skewed data plots” shown in subjects like Integrated Algebra or English. Knowing the errors within the data set, this misinterpretation raises questions like “In reality, how many people got a perfect score?” and that question can be very difficult to solve.
